Implementación de Modelo de Regresión Lineal con Validación Cruzada¶

Este cuaderno muestra la implementacin de un modelo de regresin lineal simple utilizando validacin cruzada para seleccionar los mejores parmetros del modelo. Luego, el modelo se prueba en un conjunto de datos no visto (conjunto de prueba).

En esta primera seccin, cargamos todas las libreras necesarias para llevar a cabo el anlisis.

2. División del Conjunto de Datos en Entrenamiento y Prueba¶

En esta etapa, dividimos el conjunto de datos en dos partes: uno para el entrenamiento y otro para la prueba. El conjunto de entrenamiento se utilizar para ajustar el modelo, mientras que el conjunto de prueba se utilizar para evaluar el rendimiento del modelo al final del proceso.

Utilizamos train_test_split para realizar esta divisin, separando el 70% de los datos para el entrenamiento y el 30% restante para la prueba.

3. Validación Cruzada¶

La validacin cruzada es una tcnica utilizada para evaluar el rendimiento de un modelo y ajustar sus hiperparmetros. En este cuaderno, utilizamos KFold para realizar una validacin cruzada de 5 pliegues. Esto significa que el conjunto de entrenamiento se divide en 5 subconjuntos, donde en cada iteracin 4 se utilizan para entrenar y 1 para validar.

El modelo se entrena en cada pliegue, y se evala su rendimiento utilizando la mtrica F1-Score. Finalmente, seleccionamos los mejores parmetros basndonos en el mejor F1-Score obtenido durante la validacin.

# Parmetros iniciales
alpha = 0.001  # Tasa de aprendizaje
epochs = 10000  # Nmero mximo de iteraciones
tolerance = 1e-6  # Tolerancia para la convergencia
kfolds = 5  # Nmero de pliegues para la validacin cruzada

kf = KFold(n_splits=kfolds, shuffle=True, random_state=42)
best_m, best_b = None, None
best_f1 = -np.inf

# Validacin cruzada
for fold, (train_index, val_index) in enumerate(kf.split(x_train)):
    print(f"Validacin cruzada - Pliegue {fold + 1}/{kfolds}")

    X_train_fold, X_val_fold = x_train[train_index], x_train[val_index]
    y_train_fold, y_val_fold = y_train[train_index], y_train[val_index]
    specie_train_fold, specie_val_fold = specie_train[train_index], specie_train[val_index]

    m, b = 0, 0  # Reiniciar los parmetros para cada pliegue
    n_train_fold = len(X_train_fold)

    for epoch in range(epochs):
        hxi = m * X_train_fold + b
        error = hxi - y_train_fold
        costo_m = (1 / n_train_fold) * sum(error * X_train_fold)
        costo_b = (1 / n_train_fold) * sum(error)
        m -= alpha * costo_m
        b -= alpha * costo_b

        if abs(alpha * costo_m) < tolerance and abs(alpha * costo_b) < tolerance:
            print("La convergencia ha sido alcanzada en el pliegue.")
            break

    # Evaluar en el conjunto de validacin
    predicciones_val = m * X_val_fold + b
    clasificacion_specie_val = np.where(y_val_fold > predicciones_val, 1, 0)
    f1 = f1_score(specie_val_fold, clasificacion_specie_val, pos_label=1)

    print(f"F1 Score en el pliegue {fold + 1}: {f1:.2f}")

    if f1 > best_f1:
        best_f1 = f1
        best_m, best_b = m, b

print(f"Mejor F1 Score obtenido durante la validacin cruzada: {best_f1:.2f}")
print(f"Mejor Pendiente (m): {best_m}")
print(f"Mejor Intercepcin (b): {best_b}")

4. Evaluación en el Conjunto de Prueba¶

Despus de seleccionar el mejor modelo basado en la validacin cruzada, lo evaluamos en el conjunto de prueba, que no ha sido utilizado en las etapas anteriores.

Se generan predicciones utilizando los mejores parmetros encontrados, y se calculan las mtricas de evaluacin como la precisin, el recall y el F1-Score. Tambin se genera una matriz de confusin para visualizar el rendimiento del modelo.
